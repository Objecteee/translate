# 翻译项目 SDD 核心准则
1. **Spec 优先**：任何代码修改必须参考 `specs/` 目录下的 Markdown 文档。
2. **禁止 Vibe Coding**：严禁在未定义 Data Model 的情况下编写业务逻辑。
3. **接口先行**：必须先在 `data_models.md` 中定义 interface，严禁使用 any。
4. **正交隔离**：UI 逻辑（Tauri 渲染层）与 API 逻辑（翻译引擎层）必须物理隔离。
5. **UI 规范**：实现“极简”目的，浮窗失去焦点必须立即销毁，严禁冗余动画。

## 📂 规范文件定义 (Map)
在执行任务前，请根据以下职责检索 `specs/` 目录：
- **project_scope.md**: 顶层蓝图。包含技术栈、核心功能清单、非目标。所有决策不得违反此文档。
- **data_models.md**: 实体定义。所有 TypeScript 类型、接口、数据结构必须在此同步，它是编程的“第一性要素”。
- **api_contract.md**: 链接协议。刻画翻译 API 的输入、处理步骤、输出格式及错误处理。
- **ui_ux_spec.md**: 功能表现。定义极简交互流程（如快捷键监听、浮窗销毁逻辑）。
- **task_roadmap.md**: 术的执行。记录已完成和待完成的原子任务，保持专注，一次只做一件事。

## ⚙️ 协作协议
- **修改流程**：修改代码前，若涉及逻辑变更，必须先更新对应的 spec 文件，得到确认后再写代码。
- **IPO 强制检查**：实现任何函数前，先在对话中列出该函数的 Input、Process、Output。
